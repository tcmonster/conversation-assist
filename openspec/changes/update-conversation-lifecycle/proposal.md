# update-conversation-lifecycle

## Summary
- 精简会话侧边栏展示,仅保留标题以减少干扰。
- 移除静态 mock 会话,改由真实存储数据驱动并提供新建能力。
- 支持从主页面顶栏重命名会话,确保新增会话可快速标识。

## Problem
- 侧边栏仍保留更新时间标签,与当前 UI 需求不符。
- Provider 默认注入了多组 mock 数据,无法反映真实使用过程,也阻碍新建会话流程的测试。
- 缺乏新建与重命名能力,用户无法在本地创建自定义会话,导致数据结构难以验证。

## Goals
- 侧边栏列表仅展示会话标题,行为保持置顶/搜索/归档等操作。
- 会话 provider 以 localStorage 中真实数据为准,首次加载提供空列表,新建后立即持久化。
- 新建按钮能够创建空会话并自动选中; 顶栏支持编辑标题,改名后实时保存。

## Non-Goals
- 不实现批量新建或导入导出。
- 不涉及消息写入与 AI 生成逻辑。
- 不处理跨标签同步或协同编辑。

## Risks & Mitigations
- **空列表体验**: 提供清晰空态与默认选中,避免首次进入时界面空白。
- **命名冲突**: 采用可自增的默认标题(如“未命名会话 1”),并允许用户立即编辑。
- **数据兼容**: 对 legacy 数据执行一次迁移(若检测到旧 mock)或兼容解析,避免升级后报错。
